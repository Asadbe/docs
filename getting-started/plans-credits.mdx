## Plans

| Plan | Best for | Builders | Projects | Monthly credits |
| --- | --- | ---: | ---: | ---: |
| Free | Exploring Ucode and building a first prototype | 1 | 1 | 50K |
| Starter | Small teams shipping real products | 2 | 5 | 500K |
| Pro | Growing teams with higher scale | 5 | 10 | 1M |
| Enterprise | Large organizations with custom needs | Custom | Custom | Custom |

## Billing cycles

Ucode shows three billing options on the pricing page:

- **Annual** with the largest discount
- **6 months** with a mid-term discount
- **Monthly** for the most flexible billing

## What credits are used for

Credits represent usage across AI generation and platform activity. The exact usage depends on project generation, API activity, files, database usage, server functions, and other resources.

## What to watch before launch

- **Builders**: how many teammates need to create or edit projects.
- **Projects**: how many generated apps the workspace can keep active.
- **API requests**: how much traffic the backend API can handle.
- **Database size and records**: how much structured data the app can store.
- **File storage**: how many images, documents, and generated assets the project can keep.
- **Functions and microfrontends**: how much custom backend/frontend structure the app can use.
- **AI tokens**: how much generation and editing capacity remains.

Use the Analytics page inside a project to compare real project usage against the plan.

<Warning>
  Review plan limits before launching a customer-facing app. Production projects should have enough credits, API capacity, storage, and server function limits for expected traffic.
</Warning>
