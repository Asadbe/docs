The API tab is one of Ugen's strongest production features: after Ugen generates the app, it also gives the project a usable backend API and SDK code for the selected resource.

  <img src="/images/ugen-dashboard-api-language.png" alt="Ugen API tab with SDK language selector and generated backend API code" />
</Frame>

## Why it matters

Ugen does not only create frontend screens. It prepares backend access for generated tables and resources, so a team can connect another frontend, mobile app, admin panel, automation, or external service without manually designing the API from zero.

## Main areas

- **API calls / month** shows how much API capacity the project has used.
- **SDK** contains generated client code for the selected resource.
- **API Keys** stores project keys used by external clients and integrations.
- **Integration Target** lets you choose which backend resource to generate code for, such as `users`.
- **Language selector** switches SDK examples between JavaScript, Python, Go, cURL, Dart, Kotlin, Swift, and other supported clients.
- **Generated SDK code** gives a ready request example with copy action, so developers can paste it into their app quickly.

## Tabs

| Tab | Purpose |
| --- | --- |
| SDK | Generate request/client examples for a selected backend resource and language. |
| API Keys | Create and manage keys that external clients use to call the generated backend. |

## Language selector

The language selector is important because different teams use different stacks. A web team can copy JavaScript, a backend team can use Go or Python, a mobile team can use Swift, Kotlin, or Dart, and a QA/devops team can test quickly with cURL.

## Example workflow

1. Select an integration target, such as `users`.
2. Choose the language used by your frontend, backend, or mobile team.
3. Copy the generated SDK code.
4. Add the API key from the **API Keys** tab.
5. Call the Ugen backend from your external app, admin panel, automation, or mobile client.
6. Watch API usage in Analytics and request history in Logs.

<Tip>
  This makes Ugen useful after the first generated app: the generated backend can become the data layer for other products too.
</Tip>
