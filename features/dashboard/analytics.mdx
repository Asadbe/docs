## Usage limits

The first Analytics page is the usage and plan overview. It helps the owner understand whether the project is still inside the current plan and whether the app is ready for real users.

- **Current plan** shows which workspace plan is applied to the project, such as Free, Starter, Pro, or Enterprise.
- **API calls per month** shows how many backend requests the app has used. This is important when the generated API is used by the frontend, mobile app, admin panel, or external integrations.
- **Database size** shows how much storage the generated tables and records are using.
- **Database records** shows how many rows/items are stored across project tables.
- **File size** shows uploaded and generated media usage, including images, documents, and other assets from the Files tab.
- **Monthly AI tokens** shows AI usage for the workspace month.
- **Daily AI tokens** helps track whether today's generation or editing activity is unusually high.

Use this page when a client asks if the app can handle more users, more records, more files, or more API traffic. On a Pro account, it is especially useful for confirming that the project has enough room before demo or production launch.

## Plan comparison

The plan comparison table explains what changes when the workspace upgrades or downgrades. It shows limits across plans for:

- API requests per month
- API requests per second
- Functions
- Microfrontends
- Database size
- Asset size
- Users and roles
- Items
- Tables
- API keys
- AI tokens

Review Analytics before production launch so the project has enough capacity for expected usage.

## Project Statistics

<Frame>
  <img src="/images/ugen-dashboard-analytics-project-statistics.png" alt="Ugen project statistics tab with API health and project capacity metrics" />
</Frame>

The **Project Statistics** tab shows a live technical overview of the generated project. It helps the team understand both capacity and performance.

Key metrics include:

- **Functions** shows how many server functions are used.
- **Microfrontends** shows how many generated frontend modules are active.
- **Tables** shows how many database tables exist.
- **API Keys** shows how many keys are configured.
- **Users** shows current user usage against the plan limit.
- **Database Records** shows how many records are stored.
- **Requests Per Second, Minute, and Hour** show current API traffic.
- **Average Response Time** shows backend speed.
- **Error Rate** shows how many requests are failing.
- **API Requests chart** shows request activity over time.

Use Project Statistics before demo, launch, or client handoff to confirm that the generated app is healthy and within plan limits.
