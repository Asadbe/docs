Logs show what happened inside a project. Use them to diagnose errors after generation, preview, API calls, integrations, and deployment.

## Log types

- AI generation events
- Build logs
- Runtime errors
- API request logs
- Authentication events
- Integration failures
- Server function output

## Debugging flow

1. Reproduce the issue.
2. Open logs around the same timestamp.
3. Filter by severity or request ID.
4. Trace the error to code, database rules, secrets, or integration settings.
5. Fix and retest.
