Permissions define what each user type can access inside the generated application.

<Frame>
  <img src="/images/ugen-dashboard-permissions.png" alt="Ugen permissions tab" />
</Frame>

## Permission areas

The Permissions tab includes:

- **User type selector** such as Admin
- **Table Permissions** for database-level access
- **Custom Permissions** for project-specific rules
- **Save Changes** action
- Granular permission loading for detailed rules

## Recommended workflow

1. Select the user type.
2. Review table permissions.
3. Add custom permissions for special workflows.
4. Save changes.
5. Test the app as a user with that role.
