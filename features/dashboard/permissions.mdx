title: Permissions
description: Control what builders, teammates, and app users can access.
---

Permissions protect project tools, user data, database records, files, and API actions.

## Permission layers

- **Workspace access** controls who can create and edit projects.
- **Project access** controls who can manage one project.
- **App roles** control what end users can do inside the generated app.
- **Database rules** control row and table access.
- **API rules** control endpoint access.

## Recommended roles

Start with a small role model:

| Role | Use case |
| --- | --- |
| Owner | Full workspace and billing control |
| Admin | Project and user management |
| Builder | App generation, code, database, and deployment work |
| Viewer | Read-only review access |
| App user | End-user access inside the generated product |
