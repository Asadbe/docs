The Secrets tab is the secure vault for project environment variables and sensitive information.

## Add a secret

Click **Add Secret** to open the vault secret modal.

<Frame>
  <img src="/images/ugen-dashboard-secrets-add.png" alt="Ugen add secret modal" />
</Frame>

The modal includes:

- **Key** for the environment variable name, such as `STRIPE_SECRET_KEY`
- **Environment** for selecting where the secret applies, such as Production
- **Value** for the private value
- **Create Secret** to save it into the vault

## What secrets are for

Use secrets for values such as:

```bash
STRIPE_SECRET_KEY=sk_live_...
OPENAI_API_KEY=sk-...
DATABASE_URL=postgres://...
SMTP_PASSWORD=...
```

## Actions

- Click **Add Secret** to create a new secret.
- Keep production and test values separate.
- Rotate exposed values immediately.
- Do not place secrets directly inside frontend code.

## How secrets are used

Secrets are usually consumed by server functions, integrations, API clients, and deployment configuration. They let the app connect to private services without exposing passwords or tokens in the generated frontend.

For example, SMTP credentials should be stored as secrets, then used by a server function that sends email. Payment keys, AI provider keys, database URLs, and webhook signing secrets should follow the same pattern.
