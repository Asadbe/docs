---
title: Secrets
description: Store API keys, tokens, and private environment variables.
---

Secrets are private values used by backend logic, integrations, and server functions.

## Examples

```bash
STRIPE_SECRET_KEY=sk_live_...
OPENAI_API_KEY=sk-...
DATABASE_URL=postgres://...
SENDGRID_API_KEY=SG...
```

## Rules

- Do not place secrets in frontend code.
- Use different keys for test and production environments.
- Rotate a secret immediately if it is exposed.
- Grant access only to builders who need it.

<Warning>
  Secrets should never be pasted into public prompts, screenshots, tickets, or documentation.
</Warning>
