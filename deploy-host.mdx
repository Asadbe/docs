Ugen is built for fast publishing: describe the app, generate the project, review it, then deploy and share the result.

## Deployment flow

<Frame>
  <img src="/images/ugen-dashboard-publish-share.png" alt="Ugen publish modal with share URL and app visibility settings" />
</Frame>

<Steps>
  <Step title="Review the generated project">
    Test the main user flow, confirm the UI matches the prompt, and check that data fields are correct.
  </Step>
  <Step title="Configure production values">
    Add required secrets, API keys, integrations, database rules, file storage settings, and domain settings.
  </Step>
  <Step title="Deploy">
    Use **Publish** in the project dashboard to publish the latest version.
  </Step>
  <Step title="Share and monitor">
    Open the live URL, test the production flow, then watch logs and analytics after launch.
  </Step>
</Steps>

## Share a live app

The Publish modal gives the project a shareable URL. Copy the URL and send it to a client, teammate, or tester.

You can also control **App Visibility**:

- **Public (no login)** lets anyone with the link open the app.
- Restricted visibility is useful when only invited users, employees, or clients should access the app.

After clicking **Publish App**, the latest generated version becomes available through the shared link. Use this for demos, client review, QA, and production handoff.

## Pre-launch checklist

- Authentication works for the expected user roles.
- Database tables contain the required fields.
- API keys and secrets are stored outside frontend code.
- Payment, email, and external integrations are tested.
- Logs show no blocking runtime errors.
- Analytics events are tracking the main product actions.

<Warning>
  Never launch with test API keys, exposed secrets, or unverified permissions.
</Warning>
