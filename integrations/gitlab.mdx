GitLab connects a generated project to teams that manage source code, review, automation, and deployment from GitLab.

## What this integration adds

- Links Ucode output to a GitLab repository.
- Supports developer review and continued implementation.
- Fits teams that already use GitLab projects, branches, merge requests, and pipelines.
- Keeps generated code aligned with the team's existing source control workflow.

## Setup flow

1. Open **Settings → Integrations**.
2. Select **GitLab**.
3. Click **Connect** to open the integration setup page.
4. Add GitLab authorization details.
5. Select the project, repository, and branch workflow.

## Values to prepare

- GitLab account or group access.
- Project ID or repository details.
- Access token when required.
- Target branch.

Use GitLab when deployment or review already depends on GitLab CI/CD.
