MongoDB is useful when a project needs flexible records, nested objects, or data structures that may change while the product is still evolving.

## What this integration adds

- Connects the project to document database storage.
- Supports flexible JSON-like records and nested fields.
- Helps prototypes move quickly when the schema is not final.
- Fits catalogs, content systems, user preferences, event payloads, and dynamic forms.

## Setup flow

1. Open **Settings → Integrations**.
2. Select **MongoDB**.
3. Click **Connect** to open the integration setup page.
4. Enter the MongoDB connection details.
5. Save and test data reads or writes from the generated app.

## Values to prepare

- MongoDB connection URI.
- Database name.
- Username and password if the URI does not already include them.
- Network access rules for the Ucode runtime.

Use **Logs** after setup to confirm that the project can reach the database.
