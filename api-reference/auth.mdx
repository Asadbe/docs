Ugen's Dashboard API tab generates real request examples for project tables. The generated cURL examples use the Ugen admin API host, the selected table endpoint, and project API key headers.

<Frame>
  <img src="/images/ugen-dashboard-api-curl-redacted.png" alt="Ugen Dashboard API tab showing generated cURL requests" />
</Frame>

## Base URL

```bash
https://admin-api.ucode.run/v2/items/{table}
```

For example, the `users` table uses:

```bash
https://admin-api.ucode.run/v2/items/users
```

## Header format

| Header | Value |
| --- | --- |
| `authorization` | `API-KEY` |
| `x-api-key` | Your generated project API key |
| `Content-Type` | `application/json` for create/update requests |

## Where keys come from

Create and manage keys in **Dashboard → API → API Keys**. Use one key for development/testing and a separate key for production. If a key is exposed, rotate it and update the connected app or server function.

## Security guidelines

- Keep API keys server-side.
- Use separate keys for development and production.
- Rotate leaked keys immediately.
- Scope keys to the minimum required permissions.
- Do not paste live keys into public documentation, screenshots, frontend code, or client-side repositories.
