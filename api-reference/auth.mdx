Ucode's **Settings → API** tab generates real request examples for project tables. The generated cURL examples use the Ucode admin API host, the selected table endpoint, and project API key headers.

  <img src="/images/ugen-dashboard-api-curl-redacted.png" alt="Ucode Settings API tab showing generated cURL requests" />
</Frame>

## Base URL

```bash
https://admin-api.ucode.run/v2/items/{table}
```

For example, the `research_stations` table uses:

```bash
https://admin-api.ucode.run/v2/items/research_stations
```

## Header format

| Header | Value |
| --- | --- |
| `authorization` | `API-KEY` |
| `x-api-key` | Your generated project API key |
| `Content-Type` | `application/json` for create/update requests |

## Where keys come from

Create and manage keys in **Settings → API → API Keys**. Use one key for development/testing and a separate key for production. If a key is exposed, rotate it and update the connected app or server function.

## Working request format

This is the working format generated by Ucode and verified against a live project:

```bash
curl -X GET "https://admin-api.ucode.run/v2/items/research_stations" \
  -H "authorization: API-KEY" \
  -H "x-api-key: YOUR_API_KEY"
```

Successful responses return a wrapper with `status`, `description`, and `data`.

## Security guidelines

- Keep API keys server-side.
- Use separate keys for development and production.
- Rotate leaked keys immediately.
- Scope keys to the minimum required permissions.
- Do not paste live keys into public documentation, screenshots, frontend code, or client-side repositories.
