Items are records stored inside a generated table. The **Settings → API** tab generates cURL examples for the selected table.

## List records

```bash
curl -X GET "https://admin-api.ucode.run/v2/items/research_stations" \
  -H "authorization: API-KEY" \
  -H "x-api-key: YOUR_API_KEY"
```

This returns all records from the selected table. Replace `research_stations` with any table from the Integration Target dropdown, such as `alerts`, `weather_records`, `users`, or `ice_coverage_data`.

## Get one record

```bash
curl -X GET "https://admin-api.ucode.run/v2/items/research_stations/RECORD_ID" \
  -H "authorization: API-KEY" \
  -H "x-api-key: YOUR_API_KEY"
```

Use the record `guid` from the list response as `RECORD_ID`.

## Create item

```bash
curl -X POST "https://admin-api.ucode.run/v2/items/research_stations" \
  -H "authorization: API-KEY" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "data": {
      "name": "Station Atlas",
      "station_type": "Climate Observatory",
      "latitude": "78.0000° N",
      "longitude": "15.0000° E",
      "region": "Svalbard",
      "status": "Active",
      "elevation_m": 20,
      "established_year": 2026,
      "personnel_count": 6,
      "description": "Test research station created through the Ucode API.",
      "last_transmission": "2026-05-14"
    }
  }'
```

The request body is wrapped in a `data` object. Use real column names from **Settings → Database → Table Editor** for the selected table.

## Common item actions in Settings API

- Get all records
- Get a single record by ID
- Create a new record

## Permissions

Item access depends on project permissions and API key configuration. If an item request fails, check **Settings → Logs** for the status and **Settings → Permissions** for the role/table access rules.
