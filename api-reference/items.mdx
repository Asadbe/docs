---
title: Items
description: Read and create records inside Ugen-generated tables using Dashboard API cURL examples.
---

Items are records stored inside a generated table. The Dashboard API tab generates cURL examples for the selected table.

## List records

```bash
curl -X GET "https://admin-api.ucode.run/v2/items/users" \
  -H "authorization: API-KEY" \
  -H "x-api-key: YOUR_API_KEY"
```

Replace `users` with any table from the Integration Target dropdown, such as `research_stations`, `alerts`, or `weather_records`.

## Get one record

```bash
curl -X GET "https://admin-api.ucode.run/v2/items/users/RECORD_ID" \
  -H "authorization: API-KEY" \
  -H "x-api-key: YOUR_API_KEY"
```

## Create item

```bash
curl -X POST "https://admin-api.ucode.run/v2/items/users" \
  -H "authorization: API-KEY" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"data": {"title": "New Item"}}'
```

The request body is wrapped in a `data` object. Replace the example fields with real columns from the selected table schema.

## Common item actions in Dashboard API

- Get all records
- Get a single record by ID
- Create a new record

## Permissions

Item access depends on project permissions and API key configuration. If an item request fails, check **Dashboard → Logs** for the status and **Dashboard → Permissions** for the role/table access rules.
