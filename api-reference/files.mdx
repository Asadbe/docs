Use **Settings → Files** for uploaded media, generated assets, documents, images, and project files. When generated tables include file/image fields, the API can return the stored file references as part of item records.

## File references in item APIs

For a table such as `research_stations`, a file/image column can be returned from the item endpoint:

```bash
curl -X GET "https://admin-api.ucode.run/v2/items/research_stations" \
  -H "authorization: API-KEY" \
  -H "x-api-key: YOUR_API_KEY"
```

The response can include fields that point to images or assets managed in Files, depending on the generated schema.

## Best practices

- Validate file type and size.
- Use signed URLs for private files.
- Do not expose sensitive uploads publicly.
- Keep generated folders organized by project feature or content type.

## Dashboard connection

Uploaded and generated files can be reviewed in **Settings → Files**. Use folders to keep product images, documents, reports, and user uploads organized.
