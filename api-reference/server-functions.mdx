---
title: Server Functions
description: Run backend logic that complements generated table APIs.
---

Server Functions let a Ugen project run backend logic that should not live in the browser.

## Use cases

- Payment webhooks
- Email workflows
- AI processing
- Scheduled jobs
- Data validation
- External API calls

## API relationship

The **Settings → API** tab focuses on generated table/item endpoints, such as:

```bash
https://admin-api.ucode.run/v2/items/users
```

Server functions are managed separately in **Settings → Code → Server Functions**. Use them when an action needs custom backend work before or after calling table APIs.

## Function design

Keep each function focused. A good server function accepts a small input payload, validates it, performs one job, and returns a clear success or error response.

## Dashboard connection

Server functions are managed from **Settings → Code → Server Functions**. Store private values in **Secrets**, call external services from the server function, and use **Logs** to debug execution.
